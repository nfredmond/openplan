# OpenPlan — GIS Research Handoff Protocol (Canonical)

**Date (PT):** 2026-03-02  
**Owner:** COO lane (Bart) + GIS lane (Priya)

## Purpose
Standardize GIS research updates so decisions can be reviewed quickly and traced to artifacts.

## Required Handoff Format (every update)
1. **Topic**
2. **Artifact path**
3. **Commit**
4. **Decision deltas**
5. **Risks/asks**
6. **Manual-verification items**

## Mandatory Reporting Rules
- Surface **scope-drift risk up front** in each handoff.
- If policy/license terms could not be verified by automation, list as **manual-verification item**.
- Mark lock posture explicitly (e.g., no-new-feature lock).

## Governance Notes
- Strategy artifacts do **not** authorize implementation.
- No implementation starts without explicit GO authorization under current lock posture.

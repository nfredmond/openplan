# OpenPlan First Pilot Scope Note — 2026-04-07

**Owner:** Owen Park  
**Support:** Bartholomew Hale  
**Status:** active internal scope note

## Purpose

Define the cleanest first-pilot shape for OpenPlan so the team does not drift into overbroad promises.

## Recommended first-pilot profile

The first pilot should be:
- supervised,
- scoped to one named agency or project context,
- centered on one or two proven workflows,
- and judged by usefulness and trust, not by maximum feature breadth.

## Best current in-scope workflows

Recommended first-pilot workflow options:
1. authenticated planning continuity across project / plan / model / program surfaces,
2. report traceability and source navigation,
3. scenario coordination,
4. guided county or analysis workflow where the scope is explicitly bounded.

## What the first pilot should not try to prove

Do not make the first pilot about:
- validated forecasting claims,
- broad self-serve onboarding,
- full compliance/legal automation,
- unlimited custom modeling,
- enterprise-wide replacement claims.

## Success criteria

A first pilot is successful if the agency can credibly say:
- the selected workflow completed end to end,
- the outputs were understandable,
- the support path was clear,
- the product did not overpromise,
- and the next step is obvious.

## Commercial posture

The first pilot should be described as:
- supervised,
- evidence-accurate,
- and bounded.

Use the live public pricing as the baseline reference when relevant, but be plain if the pilot is actually being handled through a custom supervised wrapper.

## Language discipline

Safe:
- production-backed
- supervised pilot
- guided workflow
- auditable continuity
- bounded scope

Unsafe:
- fully launched
- fully validated
- ready for any agency without qualification
- fully self-serve commercial product

## Recommendation

The first pilot should optimize for:
- credibility,
- repeatability,
- and a clean closeout narrative.

It should not optimize for maximum breadth.

## Bottom line

The right first pilot is a narrow, well-supported proof of value, not a heroic attempt to prove the whole OpenPlan thesis in one shot.

# OpenPlan — Engineering Research Handoff Protocol (Canonical)

**Date (PT):** 2026-03-02  
**Owner:** COO lane (Bart) + Engineering lane (Iris)

## Required Handoff Format (every update)
1. **Topic**
2. **Artifact path**
3. **Commit**
4. **Decision deltas**
5. **Risks/asks**
6. **Manual-verification items**

## Mandatory Rules
- Surface **scope-drift risk up front** in each handoff.
- Explicitly identify **P0/P1 impact** and release posture effect.
- Provide evidence paths for any reliability/security claims.
- If docs/specs/terms cannot be verified automatically, list as **manual-verification item**.
- State lock posture explicitly (e.g., no-new-feature lock).

## Governance Note
- Strategy/research artifacts do **not** authorize implementation under lock.

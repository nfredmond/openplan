## The live e2e (gap D from the quick-wins handoff — now closed)

First real AequilibraE worker run, driven end-to-end in a browser (demo user,
Playwright): launch → OSM network build (14k links) → BFW assignment
converged at rgap 0.010 → VMT KPIs → CEQA determination → signed artifact
downloads → evidence packet from private Storage. Final numbers:
`daily_vmt` 33,402, `vmt_per_capita` 0.328 (small closed network), private
`storage://run-artifacts/...` artifacts, CEQA panel rendered a determination.

It surfaced five defects no test could have caught (all fixed in #39 amend):

1. **Relaunch 500** — the requeue nulled `result_summary_json` (NOT NULL DEFAULT `{}`); reset to `{}` + audit logging + a route regression test.
2. **Census ACS API now requires a key** — returns an HTML "Missing Key" page as HTTP 200; surfaced as an actionable `DataPipelineError`. **Nathaniel action item:** free key at api.census.gov/data/key_signup.html → `CENSUS_API_KEY` in `openplan/.env.local`. Affects the worker's dynamic study areas AND live ACS in `/api/analysis`.
3. **Unbounded parallelism** — `AEQ_CORES` env (default 1) on skimming + assignment.
4. **Graph-absent centroids** — zones that can't reach the main network component crash the native engine; now excluded honestly, logged, listed in the packet (`excluded_zones`).
5. **numpy ABI heap corruption** — numpy ≥2.1 heap-corrupts aequilibrae 1.6.x's compiled path engine (glibc "corrupted size vs. prev_size", reproduced on cp311 + cp314; clean on 2.0.x). Pinned `numpy>=1.26,<2.1`.

## Worker operational notes (hard-won)

- The worker venv must be **Python 3.11 + numpy 2.0.x**. Python 3.14 source-builds aequilibrae's Cython and was one of the crash vectors; 3.11 has cp311 wheels. `~/.local/bin/python3.11 -m venv .venv311 && pip install -r requirements.txt && pip install "numpy==2.0.2"`.
- Run with `AEQ_CORES=1` on the shared/dev box (default) — bump on big machines.
- Dynamic study areas need `CENSUS_API_KEY`; without it the worker fails with a clear message. The frozen `data/pilot-nevada-county/package/` is the fallback (it needs `total_population`/`bbox` added to its manifest — the e2e did this in scratch, not committed).
- `next dev` wedged under memory pressure during the e2e; the **production server** (`npm run build && npm run start`) was rock-solid. Use it for browser verification.

## The sketch ABM lane (roadmap C–D, now launchable)

In-process, dependency-free, synchronous. Inputs built from the census + LODES
fetchers (`sketch-abm-inputs.ts`, every constant a documented screening
assumption). Measured on the real Nevada County package (26 tracts): ~3.16
trips/capita, ~9.71 VMT/capita, 82% auto. The benchmark-fit block honestly
scores this as "large deviation from reference — illustrative" (the sketch
model under-estimates absolute VMT because it models a subset of daily travel
and divides for occupancy). That honest deviation signal is the feature.

### Boundary decision (Nathaniel, 2026-07-18) — respect this

**Sketch VMT must not drive CEQA §15064.3 determinations.** Its `vmt_per_capita`
is ~56% below the CARB reference, so a determination panel would render "less
than significant" for model-artifact reasons. The `ModelRunCeqaVmtScreen` stays
gated to `engine_key === "aequilibrae"`. Sketch runs still show their VMT KPIs
+ benchmark-fit context. See memory `sketch-vmt-ceqa-boundary`.

## Known debts / next steps

- Sketch zones use a deterministic grid placement (census tract data carries no geometry); a follow-up could feed real TIGERweb centroids via `fetchTractOverlayFeatures`.
- `runABM` draws `Math.random` internally, so production trip-level output is not seed-reproducible (mode split converges within ~1pp over 2,000 households; benchmark tests stub the PRNG). Thread a seed through if per-run reproducibility is ever needed.
- Sketch tract set is county-bbox-scale (every tract of every county the corridor bbox touches), same coarseness as the deterministic lane — provenance strings say so.
- Roadmap E/F (worker select-link → resident VMT; real LODES OD + mode choice + transit LOS) remain the larger lifts, unchanged from the quick-wins handoff.
- Aerial ODM, grants.gov sync, downvotes: still v1.1-deferred, demand-driven.

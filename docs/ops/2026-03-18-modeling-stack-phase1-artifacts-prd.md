# Modeling Stack Phase 1: Staged Execution and Artifacts PRD

**Date:** 2026-03-18  
**Owner:** Iris Chen
**Status:** IMPLEMENTING

## Objective
Implement P1B.1 from the Modeling Stack Build Backlog: "Expand model run orchestration beyond thin single-run rows." This transitions OpenPlan's modeling stack from simple monolithic runs into staged orchestration necessary for the AequilibraE-first backbone.

## Requirements
1. **Model Run Stages**: We need a `model_run_stages` table to break a single run into distinct processing steps (e.g., `ingestion`, `skim_generation`, `assignment`, `kpi_extraction`).
2. **Model Run Artifacts**: We need a `model_run_artifacts` table to register external files or intermediate data outputs created during the run (e.g., `skims.omx`, `assignment_results.csv`, `network_package.zip`).

## Database Schema Additions
- `model_run_stages`:
  - `id`, `run_id` (fk to `model_runs`)
  - `stage_name` (e.g., 'ingest', 'skim', 'assignment')
  - `status` ('queued', 'running', 'succeeded', 'failed')
  - `started_at`, `completed_at`
  - `error_message`, `log_tail`
- `model_run_artifacts`:
  - `id`, `run_id` (fk to `model_runs`)
  - `stage_id` (fk to `model_run_stages`, optional)
  - `artifact_type` ('network_package', 'skim_matrix', 'assignment_results', 'kpi_summary')
  - `file_url`, `file_size_bytes`, `content_hash`

## UI Scaffolding
- Update the Model Run Detail page (`/models/[id]/runs/[run_id]`) to display:
  - A staging timeline component that shows all stages and their statuses.
  - An artifacts table or card list showing outputs generated by the run.
- Update the mock local data fetching layer to return stages and artifacts.

## Deployment Slice
A coherent slice containing the DB migration, the updated schema types, and the scaffolding UI for the model run detail view, testing securely locally and merging to `main`.

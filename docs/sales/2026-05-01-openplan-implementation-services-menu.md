# OpenPlan Implementation Services Menu

_Date: 2026-05-01_
_Status: Buyer-safe draft; scope before quoting._

## Purpose

Implementation services turn OpenPlan from software access into a working planning process. These services are scoped around the shared evidence spine: projects, plans, funding, maps, engagement, model runs, reports, and audit history.

## Service Options

### 1. Workspace Onboarding

- Confirm organization/workspace structure.
- Configure first users and roles.
- Set public branding basics.
- Confirm data sensitivity, AI/privacy posture, and support path.

### 2. Project Database Setup

- Import or structure the first project list.
- Normalize sponsor, phase, cost, funding, plan inclusion, status, and priority fields.
- Confirm shared project-spine links for RTP, grants, engagement, analysis, reports, maps, and evidence when those surfaces are in scope.
- Identify missing project evidence and next actions.

### 3. RTP / Plan Workflow Setup

- Configure RTP, ATP, LRTP, or corridor-plan workflow.
- Set horizon, scope, project list, packet/report needs, adoption milestones, and QA checklist.
- Produce a first board-ready or review-ready packet.

### 4. Grants / Funding Setup

- Configure opportunities, awards, match, milestones, reimbursement posture, and closeout tracking.
- Link funding records back to projects and reports.
- Prepare grant-readiness factors and missing-evidence list.

### 5. Engagement Handoff Setup

- Configure public intake campaign, categories, moderation path, public feedback publication, and report handoff.
- Produce an engagement summary or comment-matrix-ready packet.

### 6. Analysis / Modeling Setup

- Configure a corridor or scenario analysis workflow.
- Link model/run output to scenarios and reports.
- Add caveats and confidence boundaries.
- Do not claim calibrated/validated forecasting unless separately proven.

### 7. Data Hub / GIS Setup

- Inventory datasets, sources, vintages, licenses, geometry types, quality posture, and citations.
- Configure map layers and report-ready map/data outputs.

### 8. Aerial / Field Evidence Setup

- Scope aerial mission, imagery, field inventory, or asset evidence workflow.
- Link imagery/field records to project, grant, engagement, or report artifacts.

### 9. Report / Deliverable Templates

- Configure branded report sections, maps, tables, disclaimers, and accessibility expectations.
- Produce a first export and evidence packet.

## Pricing Posture

Quote implementation services by scope. Do not mix older historical pilot pricing with the current website service baseline unless the difference is explicitly explained.

## Required Caveats

- Human review remains required.
- AI outputs are draft/support material.
- Modeling outputs are only as strong as the configured data, assumptions, and validation.
- LAPM/legal/compliance automation is not complete unless scoped and reviewed separately.

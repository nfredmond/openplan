# OpenPlan Demo Workspace Script

_Date: 2026-05-01_
_Status: Internal guided-demo script._

## Demo Goal

Show OpenPlan as a civic planning workbench where projects, reports, grants, engagement, analysis, admin support, and billing posture share one evidence spine.

Do not demo it as a generic AI dashboard or a fully self-serve SaaS checkout product.

## 30-Minute Flow

### 0. Setup - 2 minutes

Say:

> OpenPlan is open-source planning software with managed hosting and implementation services. This demo focuses on supervised planning workflows that are already proof-backed.

Do:

- Confirm this is a guided proof-of-capability demo.
- Name the buyer's likely first workflow.

### 1. Command Center / Workspace Posture - 4 minutes

Show:

- workspace shell,
- active planning surfaces,
- evidence-first navigation,
- support/billing boundaries if relevant.

Point:

- The workspace is the shared planning record, not a disconnected dashboard.

### 2. RTP / Report Packet - 6 minutes

Show:

- RTP cycle or report registry,
- packet/detail page,
- generated HTML artifact,
- release-review/stale-basis posture.

Proof link:

- `docs/ops/2026-05-01-openplan-local-rtp-release-review-smoke.md`

### 3. Grants / Funding / Reimbursement - 5 minutes

Show:

- opportunity/award,
- project funding posture,
- reimbursement invoice,
- closeout milestone.

Proof link:

- `docs/ops/2026-05-01-openplan-local-grants-flow-smoke.md`

### 4. Engagement Handoff - 4 minutes

Show:

- public portal/share intake,
- moderation queue,
- approved public feedback,
- handoff report artifact.

Proof link:

- `docs/ops/2026-05-01-openplan-local-engagement-report-handoff-smoke.md`

### 5. Analysis / Model Run Linkage - 4 minutes

Show:

- managed run or analysis result,
- scenario attachment,
- Analysis Studio deep link,
- analysis-summary report.

Proof link:

- `docs/ops/2026-05-01-openplan-local-analysis-report-linkage-smoke.md`

### 6. Admin / Support / Billing Posture - 3 minutes

Show or summarize:

- request-access intake,
- allowlisted reviewer triage,
- provisioned workspace and owner invite path,
- billing waiver language.

Proof links:

- `docs/ops/2026-05-01-openplan-local-admin-support-flow-smoke.md`
- `docs/ops/2026-05-01-openplan-billing-current-cycle-waiver-proof.md`

### 7. Close - 2 minutes

Ask:

- Which first workflow should we scope?
- Who owns review?
- What data sensitivity or procurement constraint matters?
- Is this self-hosted, managed-hosted, implementation-only, or a mix?

## Do Not Say

- "The model is validated forecasting."
- "AI makes the planning decision."
- "Billing was freshly paid-canary-proven today."
- "Public request access automatically creates a workspace."
- "This replaces legal or professional review."
